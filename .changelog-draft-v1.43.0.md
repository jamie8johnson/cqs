### Campaign summary (release headline)

- **v1.42.0 full 16-category audit campaign — 107 fresh findings + 50 carried-forward (v1.40), every P1/P2 and P3/P4 tier closed.** The 2026-06-11 audit (PRs #1737–#1795, ~35 PRs in ~36h) swept the codebase across all 16 categories with five categories run as nested leads, then fixed the entire triage: P1 data-safety and security, P2 correctness/observability, and the P3/P4 diagnostic tail. Headline outcomes: a per-request daemon cache layer (vector index, file_set, notes, and cross-project context cached across queries — ~400ms CAGRA load and full reference-store re-merges eliminated per request); generation-stamped HNSW sidecars with honest dirty self-heal and discarded stale rebuilds; a store/search/serve schema-ownership boundary refactor (one schema owner, scoring inverted out of `store/`); crash-safe fingerprint stamping on both the CLI and watch pipelines; daemon↔CLI output parity (text-mode render-side, slim-envelope translation, stale-origin warnings, clap-derived arg forwarding); and index-time hardening against poisoned inputs (non-finite embeddings, truncated blobs, deep parse trees, Unicode-unsafe SQL case mapping). Triage is the single source of truth in `docs/audit-triage.md`.

### Added

- **`cqs scout` exposes its option knobs as flags (#1779).** `--search-limit`, `--search-threshold`, and `--min-gap-ratio` (previously internal `ScoutOptions` defaults) are now settable on both the CLI and daemon surfaces.
- **Kind-fallback observability — fired events, telemetry category, entry spans (#1773).** `record_kind_fallback` emits a `tracing::info!` at the single shared core fire point (covering CLI-direct and daemon dispatch); a new `kind_fallback` telemetry event feeds a per-command `kind_fallbacks` counter that `cqs telemetry` reports in JSON + text. `deps` on a Function name without `--reverse` now emits a kind-labeled redirect instead of a silent empty forward-type result; `trace` gains `Kind::Other`/`Kind::Multiple` handling (trace-specific fallback + candidate definition sites) and symmetric target-arg kind validation.
- **Named limit caps + new daemon/backup/sparse interval knobs (#1771).** `GRAPH_LIMIT_CAP` (100), `PLACEMENT_LIMIT_CAP` (10), `IMPACT_DEPTH_CAP` (10), `ONBOARD_DEPTH_CAP` (5) replace duplicated `clamp(1, N)` literals across the graph/info/search command + batch-handler sites. New env knobs (defaults unchanged): `CQS_BATCH_STALENESS_CHECK_MS` (100ms), `CQS_BATCH_AUDIT_RELOAD_SECS` (30s), `CQS_BATCH_CONFIG_RELOAD_SECS` (5min), `CQS_MIGRATE_KEEP_BACKUPS` (3; 0 = prune all), `CQS_LOAD_SPARSE_BATCH` (1000), plus a `CHUNKS_PAGED_MAX_LIMIT` (10k) ceiling. README env table updated.
- **Walk DoS rails (#1769).** `enumerate_files_iter` gains `MAX_WALK_DEPTH` (64) and `MAX_WALK_FILES` (500k) with `CQS_WALK_MAX_DEPTH` / `CQS_WALK_MAX_FILES` overrides and a one-shot truncation warn; README rows added.

### Fixed

- **Daemon caches vector index / file_set / notes / cross-project context across queries (DS cluster, PRs #1739, #1770).** The HNSW/base-HNSW/file_set/notes builders and the cross-project context (opened reference stores + per-store call graphs, rebuilt at all 5 cross-project graph dispatch sites) become `Arc<Mutex<Option<Arc<T>>>>` write-back cells shared between `BatchContext` and `BatchView`, following the `splade_index_cell` pattern — subsequent checkouts snapshot instead of rebuilding (~400ms CAGRA load and ~64MB×N reference re-mmaps eliminated per request). An invalidation epoch (`AtomicU64`) guards every publish so a value built from a pre-invalidation store snapshot is discarded; `read_cell_if_current` epoch-gates every fallback read; a sticky per-slot `pending_invalidation` bitmask retries deferred slots via clear-only (no epoch re-bump). Cross-project staleness has three independent rebuild triggers (local reindex, `cqs ref update` mtime/size mismatch, references-config fingerprint change). Reference stores opened with `open_readonly_small` (16MB vs 64MB mmap) to bound virtual-address reservation across a cached session.
- **Generation-stamped HNSW sidecars — honest dirty self-heal, debris-tolerant `.bak` guard, stale rebuild saves discarded (#1754).** `HnswMeta` carries a store-state stamp (`chunk_count` + `splade_generation`, the discriminator pair CAGRA already uses); the dirty-flag self-heal now requires `verify_hnsw_current` (checksums AND stamp == live store), so a complete *previous-generation* set no longer "heals" the exact crash the flag exists for. The stale-`.bak` guard moves under the exclusive save lock (no TOCTOU against a concurrent saver's transient `.bak`s): post-success debris is deleted, but a hard refusal remains when the live set is absent/invalid (the `.bak`s are then the only good copy). `save_stamped` re-reads the live stamp under the lock and discards the save when the store moved past the build snapshot (a foreign `cqs index` intervened), leaving the dirty flag set. Enrichment's `update_embeddings_with_hashes_batch` now bumps the store-wide write generation in its transaction (an embedding-only rewrite previously left the stamp matching, so the self-heal served pre-enrichment vectors); same-cycle pairs merge into the pending rebuild before drain. The rebuild thread gets a shutdown `AtomicBool` so process exit no longer kills a save mid-promote. Legacy sidecars (no stamp) load fine on the clean path.
- **HNSW/CAGRA/SPLADE persistence and staging hardening (PRs #1745, #1767, #1764).** `load_with_dim` removes stale `.tmp` staging dirs only after acquiring the shared lock (pre-lock cleanup could delete an in-flight save's staging dir); `save()`'s promote loop hard-errors into rollback on a missing staged file instead of promoting a partial set and deleting the `.bak`s. `CagraIndex::load` now sweeps orphaned `.{blob}.*.tmp` artifacts left by hard-killed saves (random suffix meant they accumulated forever — the largest slot-dir artifact). `doc_writer::atomic_write` delegates to `crate::fs::atomic_replace` (temp fsync → rename → parent fsync; copy fallback only on genuine EXDEV) instead of an in-place truncate-write that could corrupt the source on a non-EXDEV rename failure. The legacy slot checkpoint DB opens via `SqliteConnectOptions::filename()` (special-character project paths drain the WAL) with a `busy_timeout`.
- **Crash-safe fingerprint stamping across both index pipelines (closes #1755; PRs #1753, #1772).** The CLI index path now stamps the full reconcile fingerprint (`source_mtime`, `source_size`, `source_content_hash`) inside the chunk-write transaction via `Store::upsert_embedded_batch` — previously only the watch daemon stamped these, so a CLI reindex under a live daemon left stale hashes that triggered spurious reconcile churn after every branch switch. A file whose chunks straddle multiple embed batches is now stamped only by the batch carrying its *last* chunk (a crash mid-file leaves it unstamped → reclassified STALE → re-indexed, rather than half-indexed with a current fingerprint). Files that now parse to zero chunks are pruned and threaded through with an empty live set instead of leaving stale chunks polluting search forever. `parser_stage` adds a batched staleness pre-filter before the rayon parse (unchanged files are never parsed; no-op incremental index drops from O(corpus) to O(changed)); `store_stage` writes each `EmbeddedBatch` in one transaction and runs the phantom prune as a single batched transaction.
- **`needs_embedding` visibility gate enforced on the live FTS leg and embedding-reuse lookups (#1740).** A shared gated `fts_match_ids` (JOIN chunks + `needs_embedding = 0`) replaces the RRF keyword leg's ungated inline SELECT, and all three by-hash embedding lookups (`get_embeddings_by_hashes`, `get_embeddings_by_canonical_hashes`, `get_chunk_ids_and_embeddings_by_hashes`) gain the gate — so zero-vector sentinel chunks from a partial `--llm-summaries` reindex can't surface through search or get laundered into a permanent "real" embedding via the reuse resolver / watch delta path.
- **Index-time hardening against poisoned inputs (PRs #1741, #1761).** The batched HNSW build and `prepare_index_data` (the sole CAGRA build feed) skip NaN/Inf and all-zero vectors with warn+skip, keeping `id_map`/data/kept-count aligned (NaN passed the old `!= 0.0` zero filter and could panic `hnsw_rs` parallel-insert workers); error when every vector is poisoned. `CagraIndex::search` rejects non-finite queries up front and returns empty, mirroring the HNSW guard, and an oversized-k path filters non-finite distances. The embedding cache pre-checks `blob.len() % 4 == 0` before the bytemuck cast (a truncated blob panicked with `OutputSliceWouldHaveSlop`). `extract_return_sql` uses `to_ascii_uppercase()` — Unicode `to_uppercase()` isn't byte-length-preserving (ﬁ → FI), so the RETURNS offset could slice a signature mid-codepoint and panic.
- **Parser stack-safety + Python signature correctness (PARSER_VERSION 1→2; #1757).** `collect_comment_ranges` and `find_type_identifier_recursive` convert from recursion to explicit work stacks (a pathologically deep parse tree — minified output, nested parens — could overflow a 2MiB rayon worker stack and SIGSEGV `cqs index`/the watch daemon). Python `UntilColon` signature extraction scans for the first bracket-depth-0 colon (`find_def_colon`) instead of `content.find(':')`, so `def f(x: int) -> bool:` no longer truncates to `def f(x` and loses the param list / return annotation. PARSER_VERSION bumps so byte-identical Python re-extracts signatures on the next index (the stored signature refreshes; the embedding does not, since reuse keys on `canonical_hash`). `find_test_matches` predecessors gain a `None`-root sentinel so an anonymous caller no longer collides with `String::new()` and truncates the chain, plus a `bfs_max_nodes()` cap.
- **`callers --json` and `--cross-project` callers/callees emit the object shape (#1760).** `cqs callers X --json` now emits `{name, callers, count}` (mirroring callees' `{name, calls, count}`) instead of a bare array; `--cross-project` callers/callees fold into the same object with a per-entry `project` field instead of flipping to a flat `Vec`. The duplicated cross-project retrieval+truncate logic across callers/callees/impact/test-map/trace is extracted into surface-agnostic `*_cross_core` functions both surfaces call, pinned by new parity tests. `AuditModeArgs`/`AuditModeState` derive `Deserialize`; parameterless cores take empty `*Args` structs. **Breaking JSON (no external users):** `callers --json` bare-array → object.
- **Kind-fallback definitions carry `injection_flags` and `trust_level` (#1742).** `chunk_to_definition_value` (the shared transform behind every CLI graph kind-fallback and the daemon dispatch path) now emits the SECURITY.md trust signals with the search path's skip-when-default convention (injection detection runs on full content before the per-entry byte cap). Kind fallbacks serve only the local store, so the reference-code tier never applies.
- **`serve` chunk responses carry trust signals (#1761).** `ChunkDetail` emits skip-when-default `trust_level` (`vendored-code`) and a skip-when-empty `injection_flags` array, matching the search serializer; injection detection runs on full content before the 30-line preview truncation, so a directive past the cap still flags. Node/Edge entries relay name/file only and need no signals.
- **Daemon text-mode invocations render through the command renderer (#1762).** Daemon-forwarded text-mode commands (e.g. `cqs read <file>`) printed the structured JSON payload instead of the prose the CLI-direct path renders — output depended on whether a daemon happened to serve the query. `try_daemon_query` now gates on `daemon_invocation_is_json` (via a new `Commands::effective_output_format`), keeping text-mode invocations on the CLI render path; JSON mode still forwards (the fast path). An exhaustiveness test pins that every daemon-capable variant reports an output format.
- **Daemon-served searches warn on stale origins like CLI-direct (#1752).** `dispatch_search` runs the cheap per-origin `check_origins_stale` over result origins (O(result_count) mtime stats, honoring `--no-stale-check`) and attaches the sorted stale set as a skip-when-empty `_meta.stale_origins` key; `write_json_line` lifts it onto the envelope merged with the process-level worktree meta, and the CLI client prints the same stderr warning CLI-direct emits.
- **Daemon same-invocation parity — clap-derived arg translation + slot/env bypass (#1746).** `translate_cli_args_to_batch` takes a `CliArgSpec` derived from the live clap definition, so `blame -n` stays `--commits`, `-v <cmd>` / `--rrf <cmd>` no longer hard-error daemon-up, and bare queries forward search knobs spelling-for-spelling. The daemon-forward gate bypasses when a non-empty `CQS_SLOT` is set (the daemon serves its startup slot); transport failures after the socket exists promote `debug!`→`warn!` with a systemctl hint. A new `daemon_capable_variant_names()` + test caught `Affected` marked daemon-capable with no batch handler (reclassified CLI). One shared `SEARCH_LIMIT_CAP` read by both the CLI dispatcher and the batch search handler.
- **CLI translates the daemon's slim envelope to the bare surface shape (#1733).** Since the v1.40.0 SNR split, CLI output shape depended on whether a daemon served the query (bare payload in-process vs. the slim `{"data": ...}` batch envelope printed verbatim through the daemon-forward path). Client-side `classify_slim_envelope` + `daemon_payload_to_cli_text` now render daemon output as the in-process CLI would (bare + `_meta` splice under V2Bare, full v1 envelope under `CQS_OUTPUT_FORMAT=v1`, trailing-newline parity); slim error envelopes now exit non-zero.
- **Deletion-only hunks map to their containing function; staleness catches backward-mtime divergence (#1744).** `map_hunks_to_functions` probes a `count==0` deletion-only hunk as a point at the deletion seam instead of skipping it, so `-U0` pure deletions surface callers/tests/risk in `impact`/`review`/`ci`. `list_stale_files`/`check_origins_stale` replace the strict `current > stored` mtime predicate with `FileFingerprint::matches` under `MtimeOrHash` — files restored with preserved older timestamps (rsync -t, tar, robocopy, backup restores) now report stale, agreeing with the watch-loop reconcile; populated-fingerprint rows get a content-hash tiebreak so identical-byte mtime flips stay fresh.
- **`ef_search` config reaches direct-CLI index builds; index scores reused (#1758).** `build_vector_index_with_config` and `build_base_vector_index` default `ef_search` from the project config when the caller passes `None` (extracted to a pure `resolve_ef_search` seam: explicit override > config > loader default), so the documented knob takes effect on direct-CLI search paths, not only batch/daemon. `search_filtered_with_index` reuses the index-returned cosine scores as the fused base when the backend reports `index_scores_are_cosine()` (HNSW Cosine only; CAGRA gated off), skipping a redundant embedding-BLOB fetch + cosine recompute — no ranking change.
- **Audit small-P1 batch — error visibility + hot-path trims (#1743).** `load_audit_state` warns on non-NotFound read errors (EACCES/EIO) before degrading; `resolve_reuse` warns on non-finite global-cache embeddings with a `cqs cache clear` hint; `fetch_candidates_by_ids_async` skips the embedding BLOB column when scoring from pre-fused hybrid scores (~450 BLOB reads/query saved); the truncation-disabled tokenizer is cached in a `Mutex<Option<Arc<..>>>` (cleared on model swap) instead of deep-cloning the 262k-entry vocab per call.
- **Notes write-side hardening (#1748).** `rewrite_notes_file` checks the serialized output against `CQS_NOTES_MAX_FILE_SIZE` before the tmp-file write (a single oversized mutation could brick every subsequent notes op); mentions are capped (`MAX_NOTE_MENTIONS` 50, `MAX_NOTE_MENTION_BYTES` 256); note text is trimmed before validation and whitespace-only text/queries are rejected (a whitespace-only stored note acted as a cross-note wildcard); exact post-trim duplicate adds are rejected (`NoteError::Duplicate`).
- **Store prune/backup/slot/convert integrity (#1759).** A shared `delete_origins_in_tx` (FTS + chunks + function_calls + orphan sparse_vectors) routes all three wholesale-prune paths, closing the `function_calls` divergence (gitignore-driven prunes left ghost caller rows until the next `prune_all`). Backups snapshot via `VACUUM INTO` (a stable transaction) instead of `wal_checkpoint` + `fs::copy` (a concurrent commit could tear the copy). `slot list` gains a corrupt-but-openable warn ladder; `convert --overwrite` stages to a temp + `atomic_replace` instead of a bare truncating `fs::write`.
- **One HTTP trust model + tracked daemon writes + unified env-knob parsing (#1765).** `LlmConfig::resolve` is the sole cleartext-HTTP gate (any `http://` base requires `CQS_LLM_ALLOW_INSECURE=1`), deleting a dead contradictory re-check in `LocalProvider::new` that dropped the auth header (auth-requiring local vLLM would 401). The 7 `let _ = response-write` sites in the daemon dispatch path gain `write_ok_tracked`/`write_err_tracked` (warn + bump error_count). Every env-knob site routes through `cqs::limits::parse_env_*` (uniform warn-on-invalid); the private `cli/limits.rs` pair is deleted.
- **Tier-1 data-safety smalls (#1764).** V9FS (WSL2 DrvFS) + FUSE magic numbers added to the coarse-mtime statfs table (manually mounted drvfs/sshfs shares get 2s rounding); SQLITE_BUSY behavior pinned (two-connection `BEGIN IMMEDIATE` surfaces a recognizable locked error under `CQS_BUSY_TIMEOUT_MS=0`); CAGRA `save_inner` dedup kills a ~75-line twin.
- **Single-serialization daemon response path (#1766).** Socket success/error responses serialize once into a `Vec<u8>` and emit with a single `write_all`, replacing `writeln!("{}", value)` whose `Value` Display drove one write syscall per JSON fragment on the unbuffered `UnixStream` (hundreds of syscalls per multi-KB response). Wire bytes unchanged (alphabetized, single-line, newline-terminated).
- **`task`/BFS perf — single test-reachability computation, interned BFS keys (#1767).** `task_core` computes `test_reachability` once and threads it into both the scout phase and `compute_risk_and_tests`, eliminating a duplicate forward BFS; `test_reachability` keys visited/queue/counts by `Arc<str>` cloned from graph keys (matching the other BFS walkers), dropping 2-3 String allocs per node on a path that runs on every scout/task/health/review/suggest.
- **Macro-invocation scan batched (#1769).** `filter_invoked_macros` collapses from one GLOB-per-macro query (N+1 full content scans on every `cqs dead`/`health`) into a single rowid-paged content pass in one read transaction; behavior identical, with an auditable drop-count debug log.
- **Serde string-callback references count as live (#1573 tier 3b; #1787).** A bounded content scan (`filter_serde_callbacks`) drops Rust function candidates named as serde attribute callbacks (`default`/`with`/`serialize_with`/`deserialize_with`/`skip_serializing_if`/`getter = "path::fn"`) — derive-generated (de)serializers call these but the string reference is invisible to the call-graph walker, so they showed as dead. 7 production functions drop out of confident-dead.
- **`--no-auth --bind 0.0.0.0 --open` banner keeps the token out of logs (closes #1777/#1778/#1785; fix-trivial-archeo2 / not yet in main).** The serve banner prints the tokenized URL only when stdout is a TTY; on a non-TTY stdout it prints the token-free URL plus a hint that the token is per-launch and terminal-only, keeping the session secret out of journald/container logs.
- **Nightly doctest crash (#1751).** rustdoc maps ```ignore``` blocks to `#[ignore]` doctests, so the nightly's combined `cargo test -- --include-ignored` force-compiled the `ScoreSignal` doc example (which references `pub(crate)` items and can't compile as a doctest); the nightly now runs `#[ignore]`d unit/integration tests and doctests as separate steps and the example block is plain text.

### Internal

- **Store/search/serve schema-ownership boundary refactor (#1795).** `serve/data.rs` no longer runs raw sqlx against the store schema — all five builders' query bodies move into `store/serve_queries.rs` as typed-row methods, leaving serve with only wire-shaping (injection flags, trust level, preview truncation, graph construction). RRF fusion (`rrf_fuse`/`rrf_fuse_n`/`rrf_k`/`set_rrf_k_from_config`) moves out of `store/search.rs` into `search/scoring/fusion.rs`; the ASCII name-match helpers move to `store/helpers/scoring.rs`. `store/search.rs` now imports nothing from `crate::search`, killing the knob / `BoundedScoreHeap` / `rrf_fuse` back-edges while retaining only store-owned SQL. Pure code motion — bodies move verbatim, the v27 visibility gates and fused-score reuse path survive intact.
- **One query-preparation path — ref searches share the `query_core` prelude (#1793).** `prepare_query` extracts the full search prelude (classification, NameOnly-FTS-first, centroid α-floor, filter parse, SPLADE α resolution + encode, base-index selection) returning `Prepared::ShortCircuit` or `Prepared::Dense`; `query_core` and both `--ref`/`--include-refs` paths consume it. A multi-store seam on `SearchCtx` (`references()`/`reference_by_name()` returning `Arc<ReferenceIndex>` — daemon LRU, CLI per-call) lets the ref fan-out consume the same `PreparedQuery` the single-store path does. Net −324 lines; dead `cmd_query_project`/`QueryContext`/`RefQueryContext`/`cmd_query_ref_only`/`cmd_query_name_only` deleted. **Deliberate behavior change:** CLI `--ref` scoped search now classifies (SPLADE routing / centroid) unless explicit `--rrf`/`--rerank`, converging onto the daemon form; CLI `--include-refs --tokens` packs once after merge (the daemon's form) instead of double-packing.
- **`Kind`/`KindResolution` split + envelope/API renames (#1792).** `Kind` splits into `Kind` (5 routing kinds) + `KindResolution` (`Resolved`/`Ambiguous`/`Multiple`/`NotFound`), removing dead aggregate match arms. Wire-envelope `OutputFormat` → `EnvelopeShape` (the CLI `--format` flag enum keeps `OutputFormat`); `Store::lookup_by_name` → `get_chunks_by_name`; `CQS_DEFERRED_FLUSH_INTERVAL` → `CQS_DEFERRED_FLUSH_BATCHES` (it counts batches, not duration). `emit_json_error_with_data` now honors `EnvelopeShape` (slim under V2Bare) instead of always emitting the full v1 envelope. `detect_kind_for_store` returns the full `ChunkSummary` rows it read so a kind-mismatch fallback renders from the same single read that drove the routing decision (closes the reindex-between-reads drift window in `detect_fallback`/`deps`/`trace`). The WAL-on-9P finding (PB-V1.40-9) was refuted — no code change; the live index runs WAL on a v9fs mount serving multi-process with no SHMMAP failure.
- **Audit grab-bag — dead-wrapper cleanup, observability nits, platform guards (#1779).** Deletes the production-dead `gather_cross_index` wrapper, CAGRA `build_from_store` (zero callers), the never-read `CommandContext.project_cqs_dir`, and the `upsert_chunks_and_calls`/`needs_reindex` orphans; demotes CAGRA `build`/`build_with_metric` to `cfg(test)`. Adds Store corruption-UX panic-safety regression tests, `emit_json_error` adversarial round-trips, and `meta_json_fragment` `.expect`-to-warn hardening. Unifies `is_under_wsl_automount` onto `is_wsl_drvfs_path` (guarding the UNC arm to WSL/Windows hosts), strips Windows verbatim prefixes in `worktree_name`, and switches `lookup_by_name`'s debug span to record `name_len` instead of the full user-supplied name.
- **Trivial sweep — stale deny ignores, unused derives, kind doc (closes #1777/#1778; fix-trivial-archeo2 / not yet in main).** Prunes stale `RUSTSEC-2024-0388`/`RUSTSEC-2024-0384` ignores from `deny.toml` (both crates left Cargo.lock); drops unused `Serialize` derives + orphaned serde attrs from `CrossProjectCaller`/`CrossProjectCallee` (the cross cores remap field-by-field); corrects the stale `kind.rs` module-doc.
- **Happy-path + adversarial test backfill (PRs #1763, #1773, #1787).** Seeded-store coverage for the scout chain, `search_hybrid`, the flagship `cqs "<query>"` path, populated review/affected branches, every-session command spawns, and V2Bare payload shape; GPU-gated CAGRA adversarial pins (non-finite query, degenerate build vector, oversized-k sentinel filter); a slow-tests-gated end-to-end `enrichment_pass` test; classifier table-driven pins over `ChunkType::ALL` and `lookup_by_name` adversarial-input pins.
